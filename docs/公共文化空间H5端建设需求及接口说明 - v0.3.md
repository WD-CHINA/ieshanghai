# 建设需求

## 功能描述

### 账户登录

提供账户登录功能，支持包括运营管理方、主理人等用户登录系统，后续功能都需要登录后方可使用。

### 活动排期管理

根据管理后台的设置，用户可以对文化空间场地所要举办的活动进行排期管理。

主要提供活动排期查询、活动排期新增、活动排期修改、活动预约审核、活动实录记录等功能。

其中运营管理方用户能够管理文化空间所有场地的活动排期，主理人用户只能处理自己所负责场地的活动排期。

### 场地使用管理

用户可以对文化空间各个场地的使用进行登记管理。

提供场地使用查询、场地使用申请、场地使用修改、场地使用审核等功能。

## 页面清单

### 登录页

用于账户登录。页面应包括账号（手机号/用户名）、密码输入框，以及登录按钮。登录成功后跳转至导航页。

### 导航页

导航页上提供了各大功能模块的入口，目前主要是活动排期、场地使用这两大板块的入口。

### 活动排期列表页

展示当前用户所负责场地的活动排期列表。支持按活动名称、时间、场地、是否可预约等字段进行筛选查询。列表显示活动名称、时间、场地等关键信息，并提供“详情”、“编辑”、“预约情况”的入口。

### 活动排期详情页

展示单个活动排期的详细信息，包括活动名称、时间、场地、活动介绍、图片、预约时间、可预约人数等信息。提供“返回”、“编辑”、“预约详情”、“活动实录”按钮。（仅可编辑未开始的活动）

### 活动排期提交页

用于新增或编辑活动排期。表单字段包括：活动名称、活动时间、场地、活动介绍、图片、预计参与人数、是否可预约、预约时间、可预约人数等。提交后将进行数据校验，判断是否有场地及时间上的冲突，并提示操作结果。

### 活动预约详情页

当活动为可预约时，将显示活动预约列表，显示包括预约时间、联系人、联系电话、预约人数、预约状态等信息。活动预约存在“待审核”、“已通过”、“已拒绝”、“已签到”、“未参加”等状态。

活动预约记录可以点击，点击后出现的弹出层上能看到具体的预约信息包括：预约时间、联系人、联系电话、预约人数、预约备注、预约状态等；“待审核”状态预约记录的弹出层中会有“审核”按钮，管理员或主理人可以对该预约记录执行“通过”或者“拒绝”的审核操作。

注：社区居民的活动预约申请，在审核通过后，方为有效预约，可以来现场参加活动。

### 活动实录提交页

用于新增或编辑活动实录。活动实录是指在活动完成后，管理人员对活动的实际完成情况进行记录，表单字段包括：活动实际参与人次、活动情况记录、照片上传、视频上传等。

### 场地使用列表页

展示用户已提交的场地使用申请记录。支持按场地名称、使用日期、使用状态（待审核/已通过/已拒绝）筛选查询。列表显示场地名称、使用时间、使用单位、用途、使用状态等。提供“返回”、“编辑”、“审核”按钮。（仅可编辑未开始的场地使用申请）

### 场地使用详情页

展示单条场地使用申请的详细信息，包括场地名称、使用时间、使用单位、联系人、联系电话、用途说明、使用状态等。

### 场地使用提交页

用于新增或编辑场地使用申请。表单包括：场地、使用时间、使用单位、联系人、联系电话、用途说明、预计人数、备注等。提交后将校验时间冲突，判断是否有场地及时间上的冲突，并提示操作结果。

### 场地使用审核页

空间管理员可以对各主理人提交的场地使用申请进行审核。

# 数据接口

## 接口域名

### 域名

[smelapi.ieshanghai.cn/Api](http://smelapi.ieshanghai.cn/Api)/

注：接口域名并不完全固定，会根据关联的文化空间做响应的修改。

### 使用说明

1.  所有接口的实际调用地址需要将“接口域名”与“接口名”字符串拼合后使用。
2.  所有接口都采用POST请求，除了图片上传及视频上传等特定接口，参数传递均使用json格式。
3.  所有标注为登录后使用的接口，需要将登录时返回的用户session放在请求头中，才能通过验证。
